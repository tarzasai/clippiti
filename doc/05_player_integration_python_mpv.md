# Player Integration with python-mpv

## Objective
Embed playback directly in Clippiti using python-mpv instead of launching external player binaries.

## Why python-mpv here
- stronger live-stream behavior than Qt multimedia for continuously changing HLS
- robust reconnect/cache controls
- best alignment with StreamKeeper default player assumptions (`mpv`)

## Integration model
Use a Qt widget container and bind mpv rendering context to it (X11/Wayland compatible path depending on environment).

## Required capabilities
- load session playlist URL/path (`live.m3u8`)
- play/pause
- volume set/get
- mute toggle (optional)
- observe playback state/errors
- auto-reload/reopen on transient source failures

## Session playback source
Prefer local playlist path generated by buffer pipeline. Example:
- `/tmp/clippiti/<session_id>/live.m3u8`

## Suggested mpv options baseline
- low-latency but stable profile
- cache enabled for live stream resilience
- sensible demuxer read-ahead
- no terminal spam in normal mode

Keep these configurable in app settings.

## Minimal API contract

```python
class MvpPlayerAdapter:
    def attach(widget) -> None: ...
    def open(playlist_path: str) -> None: ...
    def play() -> None: ...
    def pause() -> None: ...
    def set_volume(value_0_100: int) -> None: ...
    def get_volume() -> int: ...
    def close() -> None: ...
```

## UI controls in main window
- Play/Pause button
- Volume slider (0-100)
- Clip button
- Snapshot button to save the current frame as an image
- Record toggle
- status line (loading, live, recording, error)
- floating overlay panel anchored in a corner over the video
- panel corner controls to move it to another corner
- panel layout toggle to switch between horizontal and vertical alignment

## Snapshot behavior
- Save the current mpv frame to the configured snapshot directory.
- Use a per-session filename format so repeated captures do not collide.
- Keep snapshot capture available whenever the player is rendering a valid frame.

## Failure behavior
If mpv fails while buffer is healthy:
- attempt controlled reopen with cooldown
- keep clip and record controls available if playlist remains valid

If buffer fails:
- stop player
- disable clip and record
- present reconnect action

## Multi-instance notes
- each instance has its own mpv context and buffer path
- no shared global singleton player
